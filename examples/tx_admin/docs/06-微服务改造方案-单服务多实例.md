# tx_admin 微服务改造方案
> 版本：v2.0（单服务多实例架构）  
> 日期：2026-06-29  
> 目标：将 `tx_admin` 整体作为一个微服务，注册到 r-nacos，支持水平扩展、故障自动转移、服务自动重启

---

## 一、改造目标与架构

### 1.1 核心目标

| 目标 | 说明 |
|------|------|
| **服务注册** | 每个 `tx_admin` 实例启动时自动注册到 r-nacos，优雅下线时自动摘除 |
| **配置中心** | 将 `config.toml` 迁移到 r-nacos 配置中心，支持热更新 |
| **水平扩展** | 无状态化改造后，可启动 N 个实例，通过负载均衡分担流量 |
| **故障转移** | 实例健康检查失败，r-nacos 自动摘除，流量不再转发到故障节点 |
| **自动重启** | Docker `restart: unless-stopped` / K8s `livenessProbe` 自动重启故障进程 |
| **会话共享** | sa-token 会话存入 Redis，多实例间会话状态一致 |

### 1.2 改造后架构

```
                     ┌─────────────────────────────────────────────────┐
                     │               负载均衡 (Nginx/HAProxy)            │
                     │             健康检查: /health 端点                │
                     └────────┬──────────────┬──────────────┬──────────┘
                              │              │              │
                    ┌─────────▼────┐  ┌──────▼─────┐  ┌────▼────────┐
                    │ tx_admin    │  │ tx_admin   │  │ tx_admin   │
                    │ 实例1       │  │ 实例2      │  │ 实例3      │
                    │ :8080       │  │ :8081      │  │ :8082      │
                    │ (注册到     │  │ (注册到    │  │ (注册到    │
                    │  r-nacos)   │  │  r-nacos)  │  │  r-nacos)  │
                    └──────┬──────┘  └──────┬─────┘  └─────┬───────┘
                           │                │               │
                           └────────┬───────┴───────────────┘
                                    │
                     ┌──────────────▼──────────────┐
                     │         r-nacos              │
                     │  服务注册 + 配置中心          │
                     │  (健康检查 / 配置热更新)      │
                     └──────────────┬──────────────┘
                                    │
                     ┌──────────────▼──────────────┐
                     │    共享基础设施               │
                     │  PostgreSQL / MySQL          │
                     │  Redis (sa-token 会话)       │
                     │  S3/共享存储 (文件)          │
                     └─────────────────────────────┘
```

**与拆分方案的核心区别**：
- `tx_admin` **不拆分**，整体作为一个微服务部署多个实例
- 实例间**无状态共享**：会话存 Redis，文件走 S3/共享存储
- r-nacos 负责**服务发现**（客户端通过服务名获取可用实例列表）和**配置下发**

## 二、阶段一：服务注册到 r-nacos

### 2.1 添加依赖

```toml
# admin_api/Cargo.toml 添加
[dependencies]
reqwest = { version = "0.12", features = ["json"] }
serde_json = "1.0"
```

> 注：r-nacos 兼容 Nacos HTTP API，可直接用 `reqwest` 调用，无需额外 SDK。

### 2.2 创建服务注册模块

在 `admin_api/src/` 下新建 `nacos.rs`：

```rust
// admin_api/src/nacos.rs

use std::sync::Arc;
use std::time::Duration;
use reqwest::Client;
use serde_json::json;
use tracing::{info, warn};
use tx_di_core::{tx_comp, CompInit};
use tokio::sync::Notify;

/// r-nacos 配置
#[derive(Debug, Clone)]
pub struct NacosConfig {
    pub server_addr: String,   // "http://localhost:8848"
    pub namespace: String,     // "dev" / "prod"
    pub group: String,         // "admin-system"
    pub service_name: String,  // "tx-admin"
    pub ip: String,
    pub http_port: u16,
    pub grpc_port: u16,
}

impl NacosConfig {
    pub fn from_env() -> Self {
        Self {
            server_addr: std::env::var("NACOS_SERVER_ADDR")
                .unwrap_or_else(|_| "http://localhost:8848".into()),
            namespace: std::env::var("NACOS_NAMESPACE")
                .unwrap_or_else(|_| "dev".into()),
            group: std::env::var("NACOS_GROUP")
                .unwrap_or_else(|_| "admin-system".into()),
            service_name: std::env::var("SERVICE_NAME")
                .unwrap_or_else(|_| "tx-admin".into()),
            ip: std::env::var("SERVICE_IP")
                .unwrap_or_else(|_| "127.0.0.1".into()),
            http_port: std::env::var("SERVICE_HTTP_PORT")
                .ok().and_then(|v| v.parse().ok()).unwrap_or(8080),
            grpc_port: std::env::var("SERVICE_GRPC_PORT")
                .ok().and_then(|v| v.parse().ok()).unwrap_or(50051),
        }
    }
}

/// 服务注册插件（随应用启动自动注册到 r-nacos）
#[tx_comp(init)]
pub struct NacosRegistryPlugin {
    config: NacosConfig,
    http_client: Client,
    shutdown_notify: Arc<Notify>,
}

impl CompInit for NacosRegistryPlugin {
    async_method!(
        fn async_init_impl(ctx: Arc<tx_di_core::app::App>, token: CancellationToken) -> RIE<()> {
            let config = NacosConfig::from_env();
            let http_client = Client::new();
            let shutdown_notify = Arc::new(Notify::new());

            // 1. 注册服务实例
            let register_body = json!({
                "ip": config.ip,
                "port": config.http_port,
                "serviceName": config.service_name,
                "groupName": config.group,
                "namespaceId": config.namespace,
                "healthy": true,
                "enabled": true,
                "metadata": {
                    "grpc_port": config.grpc_port.to_string(),
                    "version": env!("CARGO_PKG_VERSION"),
                },
            });

            let url = format!("{}/nacos/v1/ns/instance", config.server_addr);
            match http_client.post(&url).json(&register_body).send().await {
                Ok(resp) if resp.status().is_success() => {
                    info!("✅ 服务注册成功: {} @ {}:{}", config.service_name, config.ip, config.http_port);
                }
                Ok(resp) => {
                    warn!("服务注册失败: status={}, body={:?}",
                          resp.status(), resp.text().await.ok());
                }
                Err(e) => {
                    warn!("服务注册请求失败: {} (r-nacos 可能未启动，继续启动应用)", e);
                }
            }

            // 2. 启动心跳任务（每 5 秒发送一次心跳）
            let cfg = config.clone();
            let client = http_client.clone();
            let notify = shutdown_notify.clone();
            tokio::spawn(async move {
                let mut interval = tokio::time::interval(Duration::from_secs(5));
                loop {
                    tokio::select! {
                        _ = interval.tick() => {
                            let beat_body = json!({
                                "ip": cfg.ip,
                                "port": cfg.http_port,
                                "serviceName": cfg.service_name,
                                "groupName": cfg.group,
                                "namespaceId": cfg.namespace,
                            });
                            let url = format!("{}/nacos/v1/ns/instance/beat", cfg.server_addr);
                            let _ = client.put(&url).json(&beat_body).send().await;
                            // 心跳失败不退出，继续下次重试
                        }
                        _ = notify.notified() => {
                            info!("心跳任务停止");
                            break;
                        }
                    }
                }
            });

            // 3. 监听关闭信号，优雅注销
            let cfg = config;
            let client = http_client;
            tokio::spawn(async move {
                token.cancelled().await;
                info!("收到关闭信号，从 r-nacos 注销...");
                let body = json!({
                    "ip": cfg.ip,
                    "port": cfg.http_port,
                    "serviceName": cfg.service_name,
                    "groupName": cfg.group,
                    "namespaceId": cfg.namespace,
                });
                let url = format!("{}/nacos/v1/ns/instance", cfg.server_addr);
                let _ = client.delete(&url).json(&body).send().await;
                info!("✅ 服务注销完成");
                notify.notify_one();
            });

            Ok(())
        }
    );
}
```

### 2.3 在 `plugin.rs` 中引用

```rust
// admin_api/src/plugin.rs 顶部添加

use nacos::NacosRegistryPlugin;
// NacosRegistryPlugin 会通过 #[tx_comp(init)] 自动初始化，无需手动调用
```

## 三、阶段二：配置中心迁移

### 3.1 配置迁移策略

**本地保留最小配置**（`config/base.toml`）：

```toml
[nacos]
server_addr = "http://localhost:8848"
namespace = "dev"
group = "admin-system"
config_data_id = "tx-admin.toml"

[service]
ip = "127.0.0.1"      # 可从环境变量 SERVICE_IP 覆盖
http_port = 8080
grpc_port = 50051
```

**r-nacos 中的完整配置**（Data ID: `tx-admin.toml`，Group: `admin-system`）：

```toml
[web_config]
port = 8080
host = "::"
timeout_secs = 30

[sa_token_config]
timeout = 86400
active_timeout = 1800
store_type = "redis"
redis_url = "redis://localhost:6379/0"

[toasty_config]
database_url = "postgresql://admin:password@localhost:5432/tx_admin"

[file_config]
backend = "s3"
s3_bucket = "tx-admin-files"
```

### 3.2 配置加载实现

```rust
// admin_api/src/config_loader.rs

pub struct NacosConfigLoader {
    server_addr: String,
    namespace: String,
    group: String,
    data_id: String,
    client: Client,
}

impl NacosConfigLoader {
    pub async fn load(&self) -> anyhow::Result<String> {
        let url = format!(
            "{}/nacos/v1/cs/configs?dataId={}&group={}&namespaceId={}",
            self.server_addr, self.data_id, self.group, self.namespace
        );
        let resp = self.client.get(&url).send().await?;
        Ok(resp.text().await?)
    }
}
```

## 四、阶段三：无状态化改造（水平扩展前提）

### 4.1 会话共享（最关键）

**问题**：sa-token 默认会话存内存，实例 A 登录后实例 B 不认识该 session。

**方案**：将会话存储改为 Redis。

```toml
# r-nacos 配置
[sa_token_config]
store_type = "redis"
redis_url = "redis://localhost:6379/0"
```

需确认 `tx_di_sa_token` crate 是否支持 Redis 持久层，如不支持需自行实现 `SaTokenDao` trait。

### 4.2 文件上传共享

**方案**：使用 S3 兼容存储（已支持 `tx_di_file` S3 后端）。

```toml
[file_config]
backend = "s3"
s3_bucket = "tx-admin-files"
s3_endpoint = "https://cos.ap-guangzhou.myqcloud.com"
```

### 4.3 定时任务多实例协调

**问题**：3 个实例都启动 Job 调度器，同一任务被执行 3 次。

**方案**：使用 Redis 分布式锁：

```rust
// 执行 Job 前尝试获取锁
let lock_key = format!("job:lock:{}", job_id);
let acquired = redis::cmd("SET")
    .arg(&lock_key).arg("1")
    .arg("NX").arg("EX").arg(300)
    .query_async(&mut redis_conn).await
    .map(|v: redis::Value| matches!(v, redis::Value::Okay))
    .unwrap_or(false);

if !acquired {
    return Ok(()); // 其他实例正在执行，跳过
}
```

## 五、阶段四：健康检查与故障转移

### 5.1 新增 `/health` 端点

在 `admin_api/src/interfaces/api/` 下新建 `health_api.rs`：

```rust
use axum::{
    Json, Router,
    routing::get,
};
use serde::Serialize;

#[derive(Serialize)]
pub struct HealthResp {
    pub status: String,
    pub version: String,
    pub db: String,
}

pub fn router() -> Router {
    Router::new()
        .route("/health", get(health_check))
        .route("/health/live", get(|| async { "alive" }))
        .route("/health/ready", get(readiness_check))
}

async fn health_check() -> Json<HealthResp> {
    // 检查数据库连通性
    Json(HealthResp {
        status: "ok".into(),
        version: env!("CARGO_PKG_VERSION").into(),
        db: "connected".into(),
    })
}

async fn readiness_check() -> &'static str {
    // 实际检查 DB 连接
    "ready"
}
```

在 `admin_api/src/interfaces/api/mod.rs` 的 `open_router()` 中注册（无需认证）。

### 5.2 r-nacos 健康检查

注册时指定健康检查端口，r-nacos 定期调用 `/health` 判断实例是否健康，不健康则自动摘除。

## 六、阶段五：负载均衡与自动重启

### 6.1 Docker Compose 部署

```yaml
# docker-compose.yml
version: '3.8'

services:
  rnacos:
    image: ghcr.io/heqingpan/rnacos:latest
    ports: ["8848:8848", "9848:9848"]
    volumes: ["rnacos_data:/io/r-nacos"]
    restart: unless-stopped

  tx_admin_1:
    build: .
    ports: ["8080:8080"]
    environment:
      - NACOS_SERVER_ADDR=http://rnacos:8848
      - SERVICE_IP=tx_admin_1
    depends_on: [rnacos, postgres, redis]
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8080/health"]
      interval: 10s
      retries: 3

  tx_admin_2:
    build: .
    ports: ["8081:8080"]
    environment:
      - NACOS_SERVER_ADDR=http://rnacos:8848
      - SERVICE_IP=tx_admin_2
    depends_on: [rnacos, postgres, redis]
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8080/health"]
      interval: 10s
      retries: 3

  postgres:
    image: postgres:16-alpine
    environment:
      POSTGRES_DB: tx_admin
      POSTGRES_USER: admin
      POSTGRES_PASSWORD: password
    restart: unless-stopped

  redis:
    image: redis:7-alpine
    command: redis-server --appendonly yes
    restart: unless-stopped

  nginx:
    image: nginx:alpine
    ports: ["80:80"]
    volumes: ["./nginx.conf:/etc/nginx/nginx.conf:ro"]
    depends_on: [tx_admin_1, tx_admin_2]
    restart: unless-stopped

volumes:
  rnacos_data:
```

`Dockerfile`：

```dockerfile
FROM rust:1.81-alpine AS builder
WORKDIR /app
COPY . .
RUN cargo build -p admin_api --release

FROM alpine:latest
COPY --from=builder /app/target/release/admin_api /usr/local/bin/
EXPOSE 8080 50051
HEALTHCHECK --interval=10s --timeout=5s --retries=3 \
  CMD wget -qO- http://localhost:8080/health || exit 1
CMD ["admin_api"]
```

### 6.2 Nginx 负载均衡配置

```nginx
# nginx.conf
upstream tx_admin {
    least_conn;  # 最少连接数策略
    server tx_admin_1:8080 max_fails=3 fail_timeout=30s;
    server tx_admin_2:8080 max_fails=3 fail_timeout=30s;
}

server {
    listen 80;
    location / {
        proxy_pass http://tx_admin;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 6.3 Kubernetes 部署（生产推荐）

```yaml
# k8s/deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: tx-admin
spec:
  replicas: 3
  selector:
    matchLabels:
      app: tx-admin
  template:
    metadata:
      labels:
        app: tx-admin
    spec:
      containers:
      - name: tx-admin
        image: tx_admin:latest
        ports: [{containerPort: 8080}, {containerPort: 50051}]
        env:
        - name: NACOS_SERVER_ADDR
          value: "http://rnacos:8848"
        - name: SERVICE_IP
          valueFrom: {fieldRef: {fieldPath: status.podIP}}
        livenessProbe:
          httpGet: {path: /health/live, port: 8080}
          initialDelaySeconds: 30
          periodSeconds: 10
          failureThreshold: 3
        readinessProbe:
          httpGet: {path: /health/ready, port: 8080}
          initialDelaySeconds: 10
          periodSeconds: 5
```

K8s 通过 `livenessProbe` 自动重启不健康的 Pod，实现故障自动恢复。

## 八、验证方案

### 水平扩展验证

```bash
docker-compose up --scale tx_admin=3
# 访问 r-nacos 控制台确认 3 个实例已注册
# 连续请求 /health，观察响应来自不同实例
```

### 故障转移验证

```bash
docker-compose stop tx_admin_1
# 等待 10 秒，确认流量不再转发到 tx_admin_1
curl http://localhost/health
```

### 自动重启验证

```bash
docker kill tx_admin_2
# 等待 5 秒，确认 Docker 自动重启了容器
docker ps | grep tx_admin_2
```
