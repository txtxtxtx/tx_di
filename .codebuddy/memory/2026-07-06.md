- 完成 `tx_di_sip` 插件架构落地：`SipTx` 共享信封 + 中间件 DI 收集 + `SipPlugin`/`SipRouter`/`SipConfig`/`SipSender` 重写。
- 修复编译错误（用户"开始实现"→"先不考虑 gb28181"）：
  - `comp.rs`/`handler.rs`/`config.rs`：将不存在的 `#[tx_comp(...)]`/`CompInit` 改为 `#[derive(Component)] + #[component(...)]`，并补 `use tx_di_core::{Component, DepsTuple}`。

  - `sender.rs`：头文件 import 从 `rsipstack::sip::typed::{CallId, ContentType, Event, Expires, MaxForwards, ...}` 改为 `rsipstack::sip::{...}`（这些不在 typed 模块）。
  - `sender.rs`：`bye`/`cancel` 的 `anyhow::context(e)` 改回 `?` + `SipErr`（与 `register` 一致，RIE = Result<(), AppError>）。
  - `lib.rs`：doc 测试中 `recorder.lock().await.unwrap()` 改为 `*recorder.lock().await`（tokio Mutex 返回 guard）。
- `cargo build -p tx_di_sip` 与 `cargo test -p tx_di_sip` 均通过（34 单测 + doc-test 全绿）。
