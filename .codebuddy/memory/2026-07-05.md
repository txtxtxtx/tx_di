#### 重要发现
- `TRAIT_IMPL_MAP` 是全局静态，跨测试调用 `BuildContext::new()` 时会持续累积 trait 条目，因此不能断言固定长度
- 生命周期钩子的自由函数名 (`app_init`, `shutdown` 等) 必须放在独立模块中以避免同名冲突
- `inner_init_fn` 在 `ComponentMeta` 中定义但未被 `BuildContext::auto_register_all()` 调用 — 这是潜在的设计缺失

#### 生命周期钩子设计发现
- `#[component(app_init)]` → 自由函数 `fn app_init(comp: Arc<Self>, app: &Arc<App>) -> RIE<()>`
- `#[component(app_async_init)]` → 自由函数 `fn app_async_init(...) -> BoxFuture<RIE<()>>`
- `#[component(shutdown)]` → 自由函数 `fn shutdown(&self)` 
- 每个钩子的自由函数名都相同 (如 `app_init`)，因此同一模块中只能有一个组件使用相同钩子
- 测试中通过 `BuildContext::build_and_run()` 验证了 init/async_init 钩子
- 通过 `App::shutdown()` 直接调用验证了 shutdown 钩子
- 通过 `App::ins_run()` 验证了完整生命周期链

#### 技术问题修复
1. 命名冲突：生命周期自由函数必须放入独立模块（`pub mod xxx_mod`）
2. `String` 不满足 `Component` trait 约束，不能用于 `store.inject::<String>()`
3. `TRAIT_IMPL_MAP` 跨测试累积 — 改用内容断言而非长度断言
4. 配置文件的 TOML 结构需要 `[section]` 头匹配 `conf = "section"`
