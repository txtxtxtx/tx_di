## README 二次纠正：构建期组件不完整

用户指出快速上手示例错误：容器 `new` 后组件「不完整」，不能使用。

- `BuildContext::new` 仅做：加载配置 + 拓扑排序 + 注册工厂；Singleton 在 `register_factory` 中**立即**调工厂（build + inner_init）并缓存实例。
- `init` / `async_init` 直到 `app.ins_run()` 才执行（`App::init` / `App::async_init`）。
- 因此**构建期 `ctx.inject::<T>()` 返回的实例只完成 `inner_init`，不完整**；组件真正「可用」是在 `ins_run()` 返回之后。
- 已修正 `tx-di-core/README.md`（快速上手 + 形态示例 + `ctx.inject` 表说明）与根 `README.md`（快速上手 + `BuildContext & App` 章节 + 阶段表），统一强调此约束。
- `tx-di-macros/README.md` 的 `is_connected()` 在 `app_init` 回调中（同步初始化阶段），属正确示范，未改。
