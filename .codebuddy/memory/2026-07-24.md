### Fix 0.1 — 消除 unsafe 零值 Arc UB 风险
- **根因**：`zeroed()` 对 `Arc<dyn Trait>` 在最新 Rust 运行时被检测到直接 abort
- **修复**：
  - `Component::build` trait 签名从 `fn build(deps: Self::Deps) -> Self` 改为 `fn build(deps: Self::Deps, store: &Store) -> Self`
  - `component_impl.rs`: TraitInjectRequired 不再用 zeroed()，改用 `inject_trait_from_store::<trait_ty>(store)` 在 build 阶段安全注入
  - `inner_init.rs`: 移除全部 TraitInjectRequired ptr::write unsafe 代码
  - `factory.rs`: 简化（传 store 给 build，不再单独注入 trait）
  - 同步更新 5 个手动 impl（test_component.rs×6, aop.rs×2, config.rs×1）
- **零 unsafe 目标达成**：inner_init 不再包含任何 unsafe 块
