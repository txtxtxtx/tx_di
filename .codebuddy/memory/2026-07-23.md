# 2026-07-23

## tx_di_file 插件优化

修复了 9 个架构/实现问题（Database 变体保留待以后实现）：

1. **list_stream 真流式** (`opendal.rs`): `operator.list()` → `operator.lister()` (OpenDAL 0.57 提供 Lister 实现 Stream)，不再一次性 collect 到 Vec
2. **create_dir 仅 Local 后端执行** (`opendal.rs`): 新增 `backend: StorageBackend` 字段，S3 等 object store 跳过无意义的 mkdir 调用
3. **From<i32> → TryFrom<i32>** (`config.rs`): 移除 `unreachable!` panic，外部输入非法值安全返回 Err
4. **content_type 透传** (`opendal.rs`): `write_stream` 的 content_type 参数通过 `writer_with().content_type(ct)` 传给 OpenDAL，不再静默丢弃
5. **Cargo feature 联动** (`Cargo.toml`): `s3` feature 正确网关 `opendal/services-s3`，移除依赖中硬编码的 `services-s3`
6. **presigned_url 区分错误** (`opendal.rs`): `ErrorKind::Unsupported` 回退公开 URL，其余错误（认证/网络）正常传播
7. **list_stream 预计算 base_url** (`opendal.rs`): `trim_end_matches` 只算一次
8. **default_storage() → AppResult** (`plugin.rs`): 用 `DefaultStorageNotFound` 错误代替 `Option`，调用方可 `?` 传播
9. **FilePlugin shutdown** (`plugin.rs`): 新增 `#[component(shutdown)]` + `fn shutdown` 回调，清空 backends DashMap
10. **OpendalStorage::new() deprecated** (`opendal.rs`): 标记废弃

同步更新了 README、lib.rs doc、例子 tx_admin。
